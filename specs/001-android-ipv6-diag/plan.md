# Implementation Plan: Android IPv6 Diagnostic Tool

**Branch**: `001-android-ipv6-diag` | **Date**: 2026-06-26 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `specs/001-android-ipv6-diag/spec.md`

---

## Summary

Build a two-component IPv6 diagnostic suite for LTE/cellular networks:
1. **Go server** — a stateless, dual-stack HTTP/HTTPS server that listens on separate IPv4 and IPv6 listeners, responds to the `/diag` endpoint with the address family and protocol of the incoming connection, and is built as a single static binary via a Makefile.
2. **Android app** — a native Kotlin + Jetpack Compose application that binds all tests to the cellular interface, runs HTTP, HTTPS, ICMP, and DNS tests against both the server's IPv4 and IPv6 addresses, detects native IPv6 and 464XLAT (CLAT), and stores up to 50 sessions locally with plain-text and JSON export.

---

## Technical Context

**Server**  
- **Language/Version**: Go 1.24 (static binary, `CGO_ENABLED=0`)
- **Primary Dependencies**: Standard library (`net/http`, `crypto/tls`, `net`) only; optionally `golang.org/x/net` v0.38.0 for HTTP/2 tuning
- **Storage**: None (stateless)
- **Testing**: `go test ./...`
- **Target Platform**: Linux amd64 / arm64 (self-hosted lab server)
- **Project Type**: CLI / web service
- **Performance Goals**: Respond to `/diag` within 50 ms under normal load
- **Constraints**: Must bind IPv4 and IPv6 on separate listeners to avoid IPv4-mapped address ambiguity
- **Build**: Makefile; `make build` → `bin/server`

**Android App**  
- **Language/Version**: Kotlin 2.1.x
- **Primary Dependencies**: Jetpack Compose BOM 2025.04.01, OkHttp 4.12.0, Room 2.6.1, kotlinx.serialization 1.7.3, kotlinx.coroutines 1.9.0, AndroidX Lifecycle 2.8.x, AndroidX Navigation Compose 2.8.x
- **Storage**: Room / SQLite, on-device only, max 50 sessions
- **Testing**: JUnit 4 + Espresso for UI; MockK for unit tests
- **Target Platform**: Android 8.0+ (minSdk 26), targetSdk 35
- **Project Type**: Mobile app
- **Performance Goals**: Full test suite completes in ≤ 30 seconds (SC-001)
- **Constraints**: All tests must be bound to the cellular `Network` object (FR-001); ICMP via `Runtime.exec` (no raw sockets without root); DNS via system resolver only (FR-005)
- **Build**: Gradle 8.11 + AGP 8.7; `./gradlew installDebug`

## Key Design Decisions (Summary)

| Decision | Choice | Reason |
|----------|--------|--------|
| Server listeners | 4 separate listeners (IPv4+IPv6 × HTTP+HTTPS) | Clean address-family detection without IPv4-mapped parsing |
| Server binary | Static `CGO_ENABLED=0` | Runs on any Linux host without libc dependency |
| Android networking | `ConnectivityManager.requestNetwork(TRANSPORT_CELLULAR)` | Only supported way to force cellular binding without root |
| ICMP on Android | `Runtime.exec("ping")` / `Runtime.exec("ping6")` | Raw ICMP sockets unavailable to non-root apps |
| DNS resolver | System resolver only (carrier-assigned) | Required to observe DNS64 synthesis on 464XLAT networks |
| DNS API | `DnsResolver` (API 29+) + `InetAddress` fallback (API 26-28) | Explicit A/AAAA record type queries needed for XLAT detection |
| UI | Jetpack Compose (single module) | Modern, minimal boilerplate; fits simple diagnostic screen flow |
| Persistence | Room, max 50 sessions | Standard Android ORM; bounded storage |
| Export | Plain text + JSON | Plain text for humans; JSON for scripted comparison |
| Server auth | None (network-level access control) | Private/internal tool; firewall/VPN boundary sufficient |
